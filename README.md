# PIFMS Server v 0.7

## Описание 

REST API сервис для управления аутентификацией и авторизацией пользователей с использованием JWT токенов, поддерживающий ролевую модель доступа и безопасное хранение данных.

## Стек технологий

- Java 21
- Spring Boot 3.5.7
- Spring Security
- PostgreSQL
- JWT
- HTTPS/SSL

## Сущности

### User (Пользователь)
- `id` - уникальный идентификатор
- `username` - имя пользователя (уникальное)
- `email` - электронная почта (уникальная)
- `password` - хешированный пароль (BCrypt)
- `roles` - набор ролей пользователя

### Role (Роль)
- `id` - уникальный идентификатор
- `name` - название роли: `role_admin` | `role_user`
- `description` - описание роли

### Product
- `id` - уникальный идентификатор продукта
- `name` - имя продукта
- `isBlocked` - признак блокировки продукта

### LicenseType
- `id` - уникальный идентификатор типа лицензии
- `name` - имя типа лицензии
- `defaultDurationInDays` - длительность по умолчанию в днях
- `description` - описание типа лицензии

### License
- `id` - уникальный идентификатор лицензии
- `code` - активационный код
- `product` - продукт лицензии
- `type` - тип лицензии
- `owner` - владелец лицензии
- `user` - пользователь, выполнивший активацию
- `firstActivationDate` - дата первой активации
- `endingDate` - дата окончания лицензии
- `blocked` - флаг блокировки лицензии
- `deviceCount` - лимит устройств
- `description` - комментарий

### Device
- `id` - уникальный идентификатор устройства
- `name` - имя устройства
- `macAddress` - уникальный идентификатор устройства
- `user` - пользователь устройства

### DeviceLicense
- `activationDate` - дата активации лицензии на устройстве

### LicenseHistory
- статусы: `CREATED` | `ACTIVATED` | `RENEWED`

### UserSession (Сессия пользователя)
- `id` - уникальный идентификатор
- `userEmail` - почта пользователя
- `deviceId` - идентификатор устройства
- `accessToken` - JWT access токен
- `refreshToken` - JWT refresh токен
- `accessTokenExpiry` - время истечения access токена
- `refreshTokenExpiry` - время истечения refresh токена
- `status` - статус сессии: `ACTIVE` | `USED` | `REVOKED`

### Login (Аутентификация)
- `username` - имя пользователя
- `password` - пароль пользователя

### Registration (Регистрация)
- `username` - имя пользователя
- `email` - электронная почта
- `password` - пароль пользователя
- `passwordConfirm` - подтверждение пароля

## API-Endpoints

### Authentication (Публичные)
- `POST /api/auth/register` - регистрация нового пользователя
- `POST /api/auth/login` - авторизация пользователя
- `POST /api/auth/refresh` - обновление access токена через refresh токен

### Role Management (Требуется роль ADMIN)
- `POST /api/auth/{userId}/assign-role/{roleName}` - выдать роль пользователю
- `POST /api/auth/{userId}/remove-role/{roleName}` - отозвать роль у пользователя

### License Management
- `POST /api/licenses` - создать лицензию, доступно только администратору
- `POST /api/licenses/activate` - активировать лицензию на устройстве
- `POST /api/licenses/check` - проверить активную лицензию по устройству и продукту
- `POST /api/licenses/renew` - продлить лицензию

## Существующие проверки и JWT токены

### Требования к паролю
- Минимальная длина: 12 символов
- Максимальная длина: 100 символов
- Обязнчален минимум один прописной символ (A-Z)
- Обязателен минимум один строчной символ (a-z)
- Обязателен минимум один цифровой символ (0-9)
- Обязателен минимум один специальный символ (!@#$%^&*()_+-=[]{};\:'"|,.<>?)

### Ограничения при регистрации
- Имя пользователя должно быть уникально
- Почта должна быть уникальна и валидна
- Новый пользователь получает роль `role_user` по умолчанию

### JWT Токены
- Access токен: используется для аутентификации запросов, время жизни 600 секунд (10 минут)
- Refresh токен: используется для получения нового access токена, время жизни 3600 секунд (1 час)
- Refresh токен может быть использован только один раз (статус переходит в USED)
- Revoked токены не могут быть использованы

### Ролевой доступ
- `role_admin` - полный доступ, может управлять пользователями и ролями
- `role_user` - базовая роль для всех новых пользователей

## Лицензионный тикет

Ответ методов активации, проверки и продления лицензии содержит:
- `ticket.serverDate` - текущее серверное время
- `ticket.ticketLifetimeSeconds` - время жизни тикета
- `ticket.activationDate` - дата активации лицензии
- `ticket.expirationDate` - дата окончания лицензии
- `ticket.userId` - идентификатор пользователя
- `ticket.deviceId` - идентификатор устройства
- `ticket.blocked` - флаг блокировки лицензии
- `signature` - ЭЦП на основе содержимого тикета
